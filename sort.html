时间效率(时间复杂度)

输入规模: 对于规模更大的输入，一般需要更长的时间
运行时间的度量单位：统计算法中每一步的执行次数 

// 计算 1^3 + 2^3 + ... + N^3
 int sum(int N) {
     int i, partialSum;

     partialSum = 0;              //  赋值语句：1个时间单位
     for (i = 1; i <= N; i++)     //  1次赋值; N+1次比较; N次自增
         partialSum += i * i * i; //  1*N次赋值；2*N次乘法； 1*N次加法

     return partialSum;           //  返回语句：1个时间单位
 }

评估算法的时间增长率(时间复杂度)

—— 减治法

今晚南亭走一波,但是我只带了买一个菜的钱

减治法

Demo: f(n) = a^n;
先求n-1: f(n) = f(n-1) * a;
先求n-2: f(n-1) = f(n-2) * a
直到a^0 → 1
排序N个序列
排好N-1个序列，再插入第N个数
排好N-2个序列，再插入第N-1个数
规律
* 将规模为n的问题 变为规模为 n-1的子问题
* 减去的规模数是一个常数(这个常数是可变的)

时间复杂度分析

                    for (i = 1; i < len; i++) {
                        temp = arr[i];
                        for (j = i - 1; j >= 0 && arr[j] > temp; j--)
                            arr[j + 1] = arr[j];
                        arr[j+1] = temp;
                    }

时间复杂度：O(N^2);

插入与选择排序

* 共同点：时间复杂度 —— O(N^2)
* 插入排序：若数组有序(1 2 3 4 5) 只进行N次比较
* 插入排序：若数组逆序(5 4 3 2 1) 需N^2/2次比较
* 插入排序：平均时间 ~ N^2/4
* 选择排序：
即使在数组有序的情况下，时间复杂度依然是O(n^2);
* 选择排序：平均时间 ~ N^2/2
更多的是选用插入排序,而不是选择排序

—— 分治与递归

非常著名而且通用的算法设计，很多经典的算法都是通过分治与递归的思想来实现

一 分而治之

将一个问题化为两个较小的子问题
f(n) = a^n
转化为求： a^(n/2) * a^(n/2)
继续转化： (a^(n/4) * a^(n/4)) * (a^(n/4) * a^(n/4)) ......

二 递归

* 汉诺塔 64层
* 斐波那契数列

① 有一个最简单的基础规则可以直接得到答案
② 另一个规则将问题转化为另一个低一级别的问题

不是所有的分治算法都一定要比蛮力法更加高效

获奖者 —— 归并排序

时间复杂度分析

                    Mergesort(A[0...n-1])
                        if n > 1
                            copy A[0...[n/2]-1] to B[0...[n/2]-1] 　// 辅助数组B
                            copy A[[n/2]...n-1] to C[0...[n/2]-1]   // 辅助数组C
                            Mergesort(B[0..[n/2]-1])      // 分治
                            Mergesort(C[0..[n/2]-1])      // 分治
                            Merge(B, C, A);               // 将排序好的子数组合并成有序数组

时间复杂度: 设时间复杂度为 —— y(n)
转化为求解：y(n) = 2y(n/2) + n → y(n) = nlog(n)
空间复杂度：辅助数组 O(N)

归并与插入

+ 优点：时间复杂度远远优越插入排序；
　　　　适合较大型数据排序

+ 缺点：额外的递归调用和空间占用使得排序数据量较小时效率不如插入排序

猜数字

两种策略
① 32 → 16 → 8 → 4...
② 1 → 2 → 3 → 4 ...

为什么策略②要远远糟糕于策略①
我们从算法的最坏情况考虑(即算法的下界)
策略②最坏情况要猜测64次才能猜对(糟糕的下界)
策略①不论坏还是好,概率均为1/2 策略①的时间复杂度: O(logN)

二分策略是不是最优的？有没有更快的呢？

WHY ? ? ?
Beacuse：每次答案只有偏大 or 偏小两个分支, 只有偏大与偏小等概率时，才不存在最坏情况

12个球一模一样的球中找出一个坏球（重 or 轻）

经典问题 —— 秤球
有12个等质量的球, 但是一个球在某一天坏掉了,坏球可能变轻,也可能变重.
用一个天平, 最少经过几次一定可以找出坏球, 并说明球是变轻了还是变重了.

12个球每个球都可能变轻或者变重 → 24种可能
策略：每次称量将球分成两堆

1 2 3 4 5 6放在天平左侧7 8 10 11 12 放在天平右侧

假设天平左倾
1 2 3 4 5 6为重 or 7 8 9 10 11 12为轻
排除了1/2的可能性

这个策略是最优的吗? Ⅰ 称量结果有三种: 左倾; 平衡; 右倾
Ⅱ 将球分成两堆,永远不会有平衡的概率;每次称量减少1/2的可能性
Ⅲ 只有每一种结果的概率是相同时, 才不会存在最坏情况
Ⅳ 最优解：将平均球分成三堆来进行称量;每次称量可以减少2/3的可能性

③ 类似于猜数字中询问数字大小，这个答案是： 是 or 否;
④ 所以排序最佳的比较是每次比较后都可以排除1/2的可能性
